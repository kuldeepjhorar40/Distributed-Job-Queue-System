Core Features

Everything you need for
distributed job processing

⚡

Redis-Backed Queue

In-memory job queuing with sub-millisecond push/pop. Supports atomic operations and Lua scripting for advanced patterns.

RPUSH · BLPOP · ZADD
🔄

Exponential Backoff

Intelligent retry with configurable jitter. Failed jobs requeue with increasing delays — 2s, 4s, 8s, 16s, 32s.

Configurable · Jitter
⚖

Priority Scheduling

Sorted-set priority queue. Critical jobs jump the queue. Fair scheduling prevents starvation across priority tiers.

1–10 priority levels
🏭

Worker Concurrency

Horizontally scalable worker processes. Each slot independently consumes jobs — no shared state, no locking.

N workers · Stateless
🗄

MongoDB Persistence

Full job lifecycle stored in MongoDB. Queryable history, status tracking, and audit trail from creation to completion.

Indexed · TTL support
📊

Real-time Dashboard

Live metrics via SSE. Monitor queue depth, worker throughput, success rate, and retry counts from one pane.

SSE · Auto-refresh

Get up and running

🚀

Quick Start

# Clone & install
git clone https://github.com/you/queueflow
cd queueflow/backend && npm install

# Start services
docker-compose up -d redis mongo
npm run dev
npm run worker
⚙

Environment Setup

# backend/.env
PORT=3000
NODE_ENV=development
MONGO_URI=mongodb://localhost/queueflow
REDIS_HOST=localhost
REDIS_PORT=6379
MAX_RETRIES=5
WORKER_CONCURRENCY=3
🧪

Test with cURL

# Submit a job
curl -X POST http://localhost:3000/job \
  -H "Content-Type: application/json" \
  -d '{"task":"send_email","priority":9}'

# Check status
curl http://localhost:3000/job/{id}
