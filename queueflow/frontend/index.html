Core Features

Everything you need for
distributed job processing

⚡

Redis-Backed Queue

In-memory job queuing with sub-millisecond push/pop operations. Supports atomic operations and Lua scripting for advanced queue patterns.

RPUSH · BLPOP · ZADD
🔄

Exponential Backoff

Intelligent retry mechanism with configurable jitter. Failed jobs requeue with increasing delays — 2s, 4s, 8s, 16s, 32s.

Configurable · Jitter support
⚖

Priority Scheduling

Sorted set-based priority queue. Critical jobs jump the queue. Fair scheduling across priority tiers prevents starvation.

1–10 priority levels
🏭

Worker Concurrency

Horizontally scalable worker processes. Each worker independently consumes jobs — no shared state, no locking overhead.

N workers · Stateless
🗄

MongoDB Persistence

Full job lifecycle stored in MongoDB. Queryable history, status tracking, and audit trail for every job from creation to completion.

Indexed · TTL support
📊

Real-time Dashboard

Live metrics via SSE. Monitor queue depth, worker throughput, success rate, and retry counts from a single pane of glass.

SSE · Auto-refresh

Get up and running

🚀

Quick Start

# Clone & install
git clone https://github.com/you/queueflow
cd queueflow/backend
npm install

# Start dependencies
docker-compose up -d redis mongo

# Start backend
npm run dev

# Start worker
npm run worker
⚙

Environment Setup

# backend/.env
PORT=3000
NODE_ENV=development

MONGO_URI=mongodb://localhost:27017/queueflow
REDIS_HOST=localhost
REDIS_PORT=6379

MAX_RETRIES=5
RETRY_DELAY_MS=2000
WORKER_CONCURRENCY=3
QUEUE_NAME=job_queue
🧪

Test with cURL

# Submit a job
curl -X POST http://localhost:3000/job \
  -H "Content-Type: application/json" \
  -d '{
    "task": "send_email",
    "priority": 9,
    "payload": {"to": "you@example.com"}
  }'

# Check status
curl http://localhost:3000/job/{id}
